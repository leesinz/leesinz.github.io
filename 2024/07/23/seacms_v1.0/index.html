万能密码登录

前面提到，在admin/login也就是后台登录页面，user参数存在sql注入漏洞，这里的校验逻辑有缺陷，可以造成万能密码登录。

首先是user参数，未过滤直接拼接，校验密码时，将password参数进行了md5编码，再进行对比，如果相同，则成功登录。

首先查看数据库，8个字段，这里进行联合注入：

先获取ry4n的md5值，再通过union select插入，就满足了md5($password)==$passwords：

user=' union select 1,2,'ry4n','9eb0337e52170124ec0854dd7e30c5cd',5,6,7,8#&password=ry4n&login=yes

成功登录后台。这里其实只需要第四个字段为md5值即可，其他任意。
